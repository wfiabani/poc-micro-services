# Projeto de Arquitetura de Software Distribuída


# Detalhes do Projeto

Esta é uma *prova de conceito*, um *laboratório de resiliência* para meus estudos de *engenharia do caos*, projeto de arquitetura de micro serviços que implementa:
- Resiliência (Resillience4J, Circuit Breaker);
- Observabilidade e Telemetria (Jaeger, Loki, Prometheus, Grafana) com dashboards customizados;
- Mensageria (Apache Kafka);
- Escalabilidade (Kubernetes);
- Cache (Caffeine, Redis);


# RoadMap

- Autenticação e autorização;
- Service mesh;
- mTLS;
- Ingress, Ingress Controller;
- API Gateway;
- Proxy reverso;
- NoSQL
- CI/CD
- Elastic Search


# Metodologia

São criadas branches à partir da master onde bugs críticos aleatórios são inseridos via IA, onde a proposta é identificar e corrigir o bug, simulando situações de cãos em ambientes críticos.
